# TC-014: 聊天实时性

## 用例描述
验证用户之间实时聊天功能

## 前置条件
- 用户A（13800000001）已登录
- 用户B（13800000002）已登录
- 两者之间存在会话

## 测试步骤

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 用户A登录 | 进入首页 |
| 2 | 用户B登录（另一浏览器） | 进入首页 |
| 3 | 用户A进入消息页面 | 会话列表显示 |
| 4 | 用户B进入消息页面 | 会话列表显示 |
| 5 | 用户A选择与用户B的会话 | 聊天面板打开 |
| 6 | 用户A发送消息：测试消息 | 消息发送成功 |
| 7 | 用户B实时接收消息 | 消息实时显示 |
| 8 | 用户B回复消息 | 回复发送成功 |
| 9 | 用户A实时接收回复 | 回复实时显示 |

## 预期结果
- 消息发送成功
- WebSocket实时推送正常
- 双方消息同步

## 通过标准
- 消息实时送达
- 无消息丢失
- UI正确更新
