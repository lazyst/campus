# E2E 测试计划

## 1. 测试目标

验证校园互助平台核心业务流程的正确性，确保用户端和管理后台的功能正常运行。

## 2. 测试范围

### 2.1 用户端功能

| 模块 | 功能点 |
|------|--------|
| 认证 | 登录、注册 |
| 论坛 | 浏览帖子、发布帖子、帖子详情、评论、点赞、收藏 |
| 交易 | 浏览商品、发布商品、商品详情、联系卖家、收藏商品 |
| 消息 | 会话列表、聊天实时性 |
| 个人中心 | 查看我的帖子、我的商品、我的收藏、资料编辑 |

### 2.2 管理后台功能

| 模块 | 功能点 |
|------|--------|
| 认证 | 管理员登录 |
| 仪表盘 | 数据统计 |
| 用户管理 | 查看用户、禁用/启用用户 |
| 板块管理 | 查看板块、创建板块、编辑板块、删除板块 |
| 帖子管理 | 查看帖子、删除帖子 |
| 商品管理 | 查看商品、删除商品 |

## 3. 测试策略

### 3.1 测试方法
- **端到端测试**：使用 Playwright 模拟用户真实操作
- **API测试**：验证后端接口正确性

### 3.2 测试环境
- 后端：http://localhost:8080
- 用户前端：http://localhost:3000
- 管理后台：http://localhost:3001

### 3.3 测试数据
- 测试账号：见下方各模块测试用例
- 测试数据通过API预置

### 3.4 通过标准
- 所有测试用例通过
- 无阻塞性bug
- 核心功能可用

4. **移动端适配**：仅测试PC端响应式布局
