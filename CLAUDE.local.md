# 开发规范


<important>禁止使用ssh、scp命令。连接远程服务器、在远程服务器执行命令、上传/下载文件到远程服务器，使用ssh-mcp mcp</important>

## 任务执行规范

完成一个任务/功能就进行一次 git commit

### 任务启动规范

1. 开发前先启动项目

2. 创建回滚条件
**执行初始 git commit**
   - 展示添加了哪些文件
   - 使用描述性 commit 消息


### 测试规范

完成任务前**必须**彻底测试

测试验证清单:  
- 代码没有错误
- build 成功
- 简单任务可通过单元测试/代码审查验证
- 对于 web 应用，**必须**通过自动化工具进行端到端浏览器验证，使用 playwright mcp
  1. 使用浏览器模拟用户操作
  2. 验证：
     - 页面正确加载
     - 用户交互产生预期结果
     - 数据正确保存/显示

测试验证通过后，使用 git commit 提交更改: 
- 展示添加/修改/删除了哪些文件
- 使用描述性 commit 消息

### 代码提交规范

**每次代码更改后必须提交**：

```bash
git add -A
git commit -m "feat: 实现用户登录功能

- 添加登录表单组件
- 集成认证API
- 添加登录测试
"
```

为什么：commit 允许回滚错误更改，恢复到工作状态。
